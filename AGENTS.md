# Agent guide — meavo-navigation

Quick orientation for AI agents working in this repo. Read this before exploring blindly.

**Cursor:** `.cursor/rules/core.mdc` is always applied. `.cursor/rules/ui.mdc` applies when editing `src/client/**`.

## What this repo does

`@meavo/navigation` is the shared top-navigation npm package for all Meavo apps: the `MeavoNavBar` header (logo, nav links, user menu, sign-out), the tool switcher dropdown, and server helpers that decide which tools a user can see. Every Meavo app (gateway, hols, assembly, sales, mrp, factory, rp, clock, tasks) renders its header from this package — a change here ships to all of them on the next version bump.

## Stack

- TypeScript strict, React 19 client components + framework-agnostic server helpers
- Next.js and the Prisma client are **peer dependencies** — this is a library, not an app
- Tailwind CSS class names only — compiled by the **consumer** app's Tailwind config, not here
- Built with tsup → ESM in `dist/`; two entry points: `.` (client, `"use client"` banner) and `./server`
- No database of its own — consumers pass their `@meavo/db` Prisma client into `getAccessibleTools`
- Distributed via git tag refs (`#vX.Y.Z`) and GitHub Packages (`npm.pkg.github.com`); `prepare` builds on git installs

## Do NOT

- Add runtime `dependencies` — react, next, and the Prisma client stay peer deps / tsup externals
- Import the Prisma client or `@meavo/db` directly — use the structural `NavigationPrisma` type instead
- Put server-only code anywhere reachable from `src/index.ts` — the whole client entry is bundled with a `"use client"` banner
- Show tools a non-admin user lacks `ToolCardAccess` for in `getAccessibleTools` — revocation in gateway Admin must hide the tool immediately
- Use `linkedAppKey` for access decisions — it is only for resolving which tool is "current"
- Add a Meavo app key in one place only — update `MeavoAppKey`, `isMeavoAppKey`, `APP_FALLBACK_LABELS`, and `MEAVO_APP_HOSTS` together
- Edit `dist/` by hand — generated by tsup; it is gitignored and rebuilt by `prepare` on install
- Change public exports without updating `package.json` `exports` **and** `tsup.config.ts` together
- Commit secrets — this package reads no env vars; `MEAVO_APP_KEY` / `GATEWAY_URL` are set by consumer apps

## Commands

```bash
npm install          # also runs `prepare` → build
npm run build        # tsup → dist/
npx tsc --noEmit     # type check (no test suite, no lint config)
```

## Conventions

1. Client components live in `src/client/`, server helpers in `src/server/`, shared types/constants at the `src/` root — keep the two entry graphs separate.
2. Visual language follows the Meavo UI standard: brand green + Tailwind `slate`, mobile-first, ≥44px touch targets (see `.cursor/rules/ui.mdc`).
3. Any public-API change (exports, props, behaviour) requires: bump `version` in `package.json` → commit → tag `vX.Y.Z` → push with tags → bump the git ref in consumer repos.
